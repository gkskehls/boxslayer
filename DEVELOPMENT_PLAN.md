# BoxSlayer 개발 계획 (Development Plan)

이 문서는 'BoxSlayer' 프로젝트의 개발 순서와 각 단계별 목표를 정의합니다. 실제 구현에 앞서 전체적인 개발 로드맵을 공유하고, 효율적인 진행을 위한 가이드라인으로 활용됩니다.

## Phase 1: 핵심 게임 플레이 및 기본 UI 완성 (완료)

**목표:** 게임의 기본적인 흐름(타이틀 -> 로그인 -> 메인 허브/전투)을 확립하고, 플레이어가 게임을 시작하고 스탯을 강화하며 전투를 경험할 수 있도록 합니다. 모든 인터페이스는 텍스트 기반의 단순한 형태로 구현합니다.

### [x] 1.1. 초기 화면 흐름 및 사용자 인증 (완료)
* **목표:** 사용자가 게임에 처음 접속했을 때의 경험을 정의하고, 기본적인 로그인 방식을 제공합니다.
* **주요 작업:**
  * [x] `App.tsx`에 `TITLE_SCREEN`, `LOGIN_CHOICE_SCREEN`, `MAIN_HUB_SCREEN` 상태 추가.
  * [x] 각 화면에 맞는 텍스트 기반 UI 구현.
  * [x] "게스트 로그인" 선택 시 `MAIN_HUB_SCREEN`으로 전환 로직 구현.
  * [x] 게임 오버 시 `resetGame` 호출 및 `MAIN_HUB_SCREEN`으로 복귀 로직 구현.

### [x] 1.2. 스탯 밸런싱 초기 적용 및 UI 개선 (완료)
* **목표:** `GAME_DESIGN.md`에 정의된 스탯 밸런싱 가이드라인을 `gameStore`에 반영하고, 스탯 강화 UI의 피드백을 개선합니다.
* **주요 작업:**
  * [x] `gameStore.ts`의 `distributeStat` 로직을 업데이트하여 각 스탯 투자 시 증가량 (`공격력 +1`, `방어력 +1`, `최대체력 +10`, `공격속도 +0.05`)을 `GAME_DESIGN.md`에 맞춰 정확히 적용.
  * [x] 스탯 강화 버튼 클릭 시, 해당 스탯이 얼마나 증가하는지 텍스트로 명확하게 표시하는 UI 개선 (예: 버튼 텍스트에 `+1 ATK` 등).

### [x] 1.3. 사용자 데이터 관리 (게스트 로그인) (완료)
* **목표:** 게스트 사용자의 게임 진행 데이터를 로컬 스토리지에 저장하고 불러오는 기능을 구현하여, 브라우저를 닫았다 열어도 진행 상황이 유지되도록 합니다. **(참고: 이 기능은 데이터베이스 준비 전까지 임시적으로 로컬 스토리지를 활용하며, 추후 Firebase 등 실제 데이터베이스 연동으로 전환될 예정입니다.)**
* **주요 작업:**
  * [x] `gameStore`에 `saveGame` 및 `loadGame` 액션 추가.
  * [x] `localStorage`를 사용하여 `GameState`를 JSON 형태로 저장/불러오기.
  * [x] 게임 시작 시 (또는 로그인 후) 저장된 데이터가 있으면 불러오고, 없으면 `initialPlayer`로 시작하는 로직 구현.
  * [x] `gameStore`의 상태 변화 시 자동으로 `saveGame`이 호출되도록 `useEffect` 또는 `subscribe` 활용.

### [x] 1.4. 화면 내비게이션 시스템 구현 (완료)
* **목표:** 게임의 다양한 화면(마을, 전투, 스탯, 코어, 상점 등) 간의 전환을 관리하는 시스템을 구축하고, `App.tsx`의 역할을 화면 라우팅 및 레이아웃 관리로 분리합니다.
* **주요 작업:**
  * [x] `App.tsx`의 `GameScreen` 타입을 확장하여 `TOWN_SCREEN`, `BATTLE_SCREEN`, `STATS_SCREEN`, `CORE_SCREEN`, `SHOP_SCREEN` 등 추가.
  * [x] `App.tsx` 내에서 `screen` 상태에 따라 각 화면 컴포넌트를 조건부 렌더링하도록 구조 변경.
  * [x] `TOWN_SCREEN` (마을 화면) 컴포넌트의 기본 레이아웃 및 내비게이션 버튼 (전투, 스탯, 코어, 상점) 구현.
  * [x] 기존 `MAIN_HUB_SCREEN`의 전투 관련 UI를 `BATTLE_SCREEN` 컴포넌트로 분리.
  * [x] 기존 `MAIN_HUB_SCREEN`의 스탯 강화 UI를 `STATS_SCREEN` 컴포넌트로 분리.
  * [x] 각 화면 컴포넌트 간 전환을 위한 액션 (`goToTown`, `goToBattle`, `goToStats` 등) 구현.

### [x] 1.5. UI/UX 개선 (초기 단계) (완료)
* **목표:** 현재 텍스트 기반의 UI에 시각적인 구분을 추가하고, 스탯 관리 화면의 사용성을 개선하며, 전역 내비게이션을 도입하여 사용자 경험을 향상시킵니다.
* **주요 작업:**
  * [x] **UI 영역 구분:** 각 화면(전투, 스탯, 코어 등) 내에서 주요 섹션들을 시각적으로 구분.
  * [x] **남은 스탯 포인트 표시:** `StatsScreen` 컴포넌트 내에 현재 플레이어의 `statPoints`를 명확하게 표시.
  * [x] **스탯 버튼 명확화:** `StatsScreen`의 스탯 증가 버튼이 클릭 가능한 버튼처럼 보이도록 스타일 개선.
  * [x] **글로벌 내비게이션 바:** 모든 게임 화면 하단에 고정된 내비게이션 바를 구현하여 화면 간 직접 이동 지원.

## Phase 2: 코어 시스템 구현 (완료)

**목표:** 플레이어 박스에 '코어'를 장착하여 특수 효과를 부여하고, 이를 통해 전투에 전략적 요소를 추가합니다.

### [x] 2.1. 코어 데이터 구조 정의 (완료)
* **목표:** `GAME_DESIGN.md`에 정의된 코어의 속성들을 TypeScript 타입으로 정의합니다.
* **주요 작업:**
  * [x] `src/types/game.ts`에 `CoreType`, `CoreEffect`, `Core` 인터페이스 추가.
  * [x] 각 코어의 속성 (이름, 타입, 레벨, 효과, 강화 비용 등) 정의.

### [x] 2.2. `gameStore`에 코어 로직 추가 (완료)
* **목표:** `gameStore`가 코어의 획득, 장착, 강화, 인벤토리 관리를 처리할 수 있도록 확장합니다.
* **주요 작업:**
  * [x] `GameState`에 `playerCores: Core[]`, `equippedCores: (Core | null)[]` 필드 추가.
  * [x] `acquireCore`, `equipCore`, `unequipCore`, `upgradeCore` 액션 로직 구현.

### [x] 2.3. 코어 관리 UI 구현 (완료)
* **목표:** 메인 허브 화면에서 플레이어가 코어를 확인하고, 장착/해제/강화할 수 있는 UI를 제공합니다.
* **주요 작업:**
  * [x] 클릭 시 코어 인벤토리, 장착 슬롯, 코어 상세 정보 및 강화 버튼을 표시하는 UI 구현.
  * [x] 코어의 특수 효과 및 강화 수치를 텍스트로 명확하게 표시.

### [x] 2.4. 코어 데이터 모델 경량화 및 동적 계산 구현 (완료)
* **목표:** 코어 데이터 객체의 경량화 및 상태 동기화.
* **주요 작업:**
  * [x] `Core` 인터페이스 단순화(id, type, level 중심).
  * [x] `getCoreStats`, `getUpgradeCost` 등 실시간 계산 함수 적용.
  * [x] 강화 즉시 UI 반영을 위한 상태 동기화 로직 구현.

### [x] 2.5. 전투 로직에 코어 효과 반영 (완료)
* **목표:** 장착된 코어의 특수 효과가 실제 전투에 적용되도록 `gameStore`의 전투 로직을 수정합니다.
* **주요 작업:**
  * [x] `attackEnemy` 및 `attackPlayer` 액션 내에서 장착된 코어 타입을 확인하여 특수 효과 연동.
  * [x] **불 코어:** 방어력을 100% 무시하는 화염 트루 피해 엔진 구축.
  * [x] **물 코어:** 매 층 리셋되는 시작 대량 쉴드 및 타격 재생, 피해 반사 메커니즘 연동.
  * [x] **바람 코어:** 누적 타격 수 기반의 확정 연격 및 확정 회피 잔상 차단 로직 구현.
  * [x] **번개 코어:** 누적 타격 수 기반의 적 기절 제어 및 무방비 상태 적 대상 트루 데미지(처형) 로직 구현.

## Phase 3: 게임 진행 및 메타 시스템 확장 (현재 진행 중)

**목표:** 게임의 진행 단계를 구체화하고, 플레이어가 장기적으로 게임을 즐길 수 있는 메타 시스템의 기반을 마련합니다.

### [x] 3.1. 시스템 안정성 및 예외 처리 강화 (완료)
* **목표:** 데이터 무결성 강화 및 타입 안정성 확보.
* **주요 작업:**
  * [x] `upgradeCore` 로직 내 골드 부족 방지 Guard Clause 구현.
  * [x] 골드 마이너스 오류 차단 및 사용자 알림(`alert`) 로직 추가.
  * [x] ESLint 경고 해결 및 타입 안정성 최적화.

### [x] 3.2. 적 박스 종류 다양화 및 보상 시스템 (완료)
* **목표:** 다양한 패턴의 적을 구현하고 처치 시 재화를 획득하도록 합니다.
* **주요 작업:**
  * [x] `gameStore.ts`의 `spawnEnemy` 로직을 개선하여 1~10층 로테이션 가중치 및 보스 스탯 적용.
  * [x] 100층 단위 체력 10배 증폭 기믹(통곡의 벽) 구현.
  * [x] `GameState`에 `gold: number` 필드 추가 및 `attackEnemy` 액션에서 골드 지급 처리.
  * [x] UI에 현재 골드 보유량 표시.

* **목표:** 환생 로직(포인트 계산 및 초기화)과 PoE 스타일의 스킬 트리 UI/엔진을 구현합니다.
* **주요 작업:**
  * [ ] `GameState`에 `reincarnationPoints: number` 및 `acquiredNodes: string[]` (활성화된 노드 ID 리스트) 필드 추가.
  * [ ] `reincarnate()` 액션 구현:
    * [x] 스테이지/레벨/코어 레벨에 따른 환생 포인트 계산.
    * [x] 플레이어 상태(스테이지, 레벨, 스탯, 골드, 코어 레벨) 초기화.
    * [ ] `reincarnationPoints` 누적 및 `acquiredNodes` 유지.
  * [ ] `SkillTree` 데이터 정의:
    * [ ] 모든 노드 정보(ID, 능력치, 연결 정보, 위치 좌표)를 담은 정적 데이터 생성.

    * [x] 환생 버튼: 현재 진행 상황 기준 획득 예상 포인트 표시.
    * [x] 트리 UI: SVG를 활용하여 노드와 연결 선을 렌더링.
    * [x] 노드 클릭 시: 포인트 소모 및 인접 노드 확인 후 활성화 로직.
  * [ ] `getComputedStats` 고도화:
    * [x] 현재 활성화된 `acquiredNodes`를 기반으로 보너스 수치를 실시간 합산하여 실제 플레이어 스탯에 적용.

## Phase 4: PvP 시스템 및 게임 확장 (추후 확장)

**목표:** 기존 4개 스탯 체계를 3대 스탯으로 리팩토링하고, 플레이어 간 경쟁 요소를 도입하며 게임의 몰입도를 높입니다.

### [x] 4.1. 스탯 체계 리팩토링 (완료)
* **목표:** 일관성 있는 스탯 관리를 위한 3대 스탯 체계 전환.
* **주요 작업:**
  * [x] `Stats` 타입 정의를 3대 스탯(STR, DEX, CON)으로 변경.
  * [x] `gameStore` 내 스탯 분배 로직을 3대 스탯 기반으로 전면 수정.
  * [x] 스탯 총합 및 비율에 따른 외형(색상/크기) 계산 유틸리티 함수 구현 (`BattleScreen` 동적 스타일링 적용).
  * [x] 모든 스탯 변경 및 전투 계산 시 실시간 계산 패턴(`getComputedStats`) 적용.

### [ ] 4.2. PvP 빌드 및 대전 시스템 구축
* **목표:** 유저 간 경쟁을 위한 비동기 1:1 대전 기능 구현.
* **주요 작업:**
  * [ ] PvP 빌드 저장을 위한 `PvPProfile` 상태 추가.
  * [ ] 1:1 대전 엔진 (`combatEngine.ts`) 분리 구현 (제한 시간, 판정, 회피 확률 등).
  * [ ] 메인 허브 화면에 "PvP 대전" 버튼 및 빌드 설정 UI 구현.
  * [ ] 상대방 빌드 정보 식별을 위한 박스 시각화 UI 컴포넌트 고도화.

### [ ] 4.3. 사운드 및 시스템 설정
* **목표:** 게임의 타격감과 몰입도를 극대화할 사운드 시스템 추가.
* **주요 작업:**
  * [ ] 배경 음악(BGM) 및 코어 속성별 전투 타격음(SFX) 에셋 연동.
  * [ ] 설정 화면에 사운드 On/Off, 볼륨 조절 UI 구현.
