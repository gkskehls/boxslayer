# BoxSlayer 게임 디자인 문서 (Game Design Document)

## 1. 게임 개요 (Game Overview)
'BoxSlayer'는 단순한 박스 형태의 캐릭터를 성장시켜 다양한 적 박스들을 물리치는 방치형 1:1 박스 배틀 웹 게임입니다. 플레이어는 레벨업을 통해 스탯을 분배하고, 이에 따라 캐릭터의 외형이 변화하는 것을 경험하며, 더 강력한 적에 도전하게 됩니다.

## 2. 핵심 게임 루프 (Core Game Loop)
1.  **전투 시작:** 플레이어 박스와 적 박스가 1:1로 자동 전투를 시작합니다.
2.  **전투 진행:** 박스 간의 '몸통박치기'를 통해 서로 데미지를 주고받습니다.
3.  **승리:** 적 박스의 체력이 0이 되면 전투에서 승리합니다.
    *   경험치 및 재화(골드 등)를 획득합니다.
    *   다음 적 박스가 등장합니다.
4.  **레벨업:** 일정 경험치 도달 시 플레이어 박스가 레벨업합니다.
    *   스탯 포인트를 획득합니다.
    *   스탯 분배를 통해 캐릭터의 외형이 변화합니다.
5.  **패배:** 플레이어 박스의 체력이 0이 되면 전투에서 패배합니다.
    *   재시도하거나, 스탯을 강화하여 다시 도전합니다.

## 3. 전투 메커니즘 (Combat Mechanics)
*   **전투 방식:** 플레이어 박스와 적 박스가 화면의 좌우에 위치하여 1:1로 자동 전투를 진행합니다.
*   **공격 방식:** '몸통박치기' (충돌)를 통해 데미지를 교환합니다.
    *   일정 주기(공격 속도 스탯에 따라 결정)마다 서로에게 데미지를 입힙니다.
*   **데미지 계산:** (공격자의 공격력) - (피격자의 방어력)을 기본으로 하며, 추가적인 요소는 추후 고려합니다.
*   **승패 조건:** 한쪽 박스의 체력이 0이 되면 해당 박스는 파괴되고 전투가 종료됩니다.

## 4. 성장 및 커스터마이징 (Progression & Customization)
*   **레벨업:** 적 박스 처치 시 경험치를 획득하며, 일정 경험치에 도달하면 레벨업합니다.
*   **스탯 포인트:** 레벨업 시 스탯 포인트를 획득하며, 플레이어는 이 포인트를 원하는 스탯에 분배할 수 있습니다.
*   **주요 스탯:**
    *   **공격력 (Attack):** 몸통박치기 시 입히는 데미지 증가.
    *   **방어력 (Defense):** 몸통박치기 시 받는 데미지 감소.
    *   **체력 (Health):** 최대 체력 증가.
    *   **공격 속도 (Attack Speed):** 몸통박치기 빈도 증가 (예: 1초에 1회 -> 0.8초에 1회).
*   **외형 변화:** 스탯 투자에 따라 플레이어 박스의 외형이 변화합니다.
    *   **공격력:** 박스의 테두리가 더 날카로워지거나 붉은색 계열로 변화.
    *   **방어력:** 박스의 두께가 두꺼워지거나 회색/금속성으로 변화.
    *   **체력:** 박스의 크기가 커지거나 내부 패턴이 견고해짐.
    *   **공격 속도:** 박스 주변에 잔상 효과 또는 번개 효과 추가.
    *   (예시이며, 구체적인 외형 변화는 구현 시 확정)

## 5. 적의 종류 (Enemy Types)
적 박스는 난이도에 따라 다음과 같이 구분됩니다.
*   **일반 박스:** 기본적인 적. 다양한 색상과 크기로 등장하며, 플레이어의 성장에 맞춰 능력치가 점진적으로 강해집니다.
*   **중간 보스 박스:** 일정 스테이지마다 등장합니다. 일반 박스보다 훨씬 강력하며, 특정 스탯(예: 매우 높은 방어력, 빠른 공격 속도)에 특화된 경우가 많습니다.
*   **보스 박스:** 각 챕터 또는 구역의 최종 보스입니다. 매우 강력한 능력치와 독특한 외형을 가지며, 처치 시 특별한 보상을 제공합니다.

## 6. 그래픽 및 UI (Graphics & User Interface)
*   **그래픽 스타일:** 단순하고 미니멀리즘한 박스 형태의 그래픽. 웹에서 가볍게 구현될 수 있도록 최적화합니다.
*   **캐릭터/적:** 기본적으로 사각형 형태를 유지하며, 스탯에 따른 외형 변화를 통해 시각적인 피드백을 제공합니다.
*   **전투 화면:**
    *   좌측: 플레이어 박스, 우측: 적 박스.
    *   각 박스 상단에 체력 바 표시.
    *   데미지 발생 시 박스 주변에 데미지 텍스트(팝업) 표시.
    *   배경은 단순한 격자 무늬(Grid) 또는 단색.
*   **UI 요소:**
    *   **스탯 분배 화면:** 레벨업 시 활성화되며, 스탯 포인트 분배 및 캐릭터 외형 변화를 미리 볼 수 있습니다.
    *   **인벤토리/장비:** (추후 확장 시 고려)
    *   **승리/패배 화면:** 전투 결과 및 획득 보상 표시.
    *   **게임 설정:** 사운드, 그래픽 옵션 등.

## 7. 기술적 고려사항 (Technical Considerations)
*   **플랫폼:** 웹 브라우저 (데스크톱 및 모바일 웹).
*   **프레임워크/라이브러리:** React, TypeScript, Vite, Tailwind CSS.
*   **게임 루프:** `requestAnimationFrame`을 활용하여 효율적인 게임 업데이트 및 렌더링을 구현합니다.
*   **상태 관리:** Zustand 또는 React Context API를 활용하여 게임 상태를 관리합니다.
*   **성능:** 가벼운 그래픽과 효율적인 로직으로 웹 환경에서 부드러운 플레이 경험을 제공합니다.
